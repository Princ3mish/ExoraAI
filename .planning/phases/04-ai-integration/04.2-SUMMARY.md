# Wave 2 Summary: AI Robustness Layers and Persistence

## What was built
Implemented critical robustness and normalization layers for the AI processing pipeline:
- **Zod Validation & Normalization**: Output schemas (`suggestTimesOutputSchema`, `draftEmailOutputSchema`, etc.) are now cleanly parsed with Zod instead of manual iteration, and automatically normalized (cleaning markdown blocks, standardizing output into ISO dates).
- **Timeouts**: Wrapped all orchestration calls inside a 15,000ms `Promise.race` timeout guard to prevent long-running hanging calls if the LLM providers stall.
- **Quota Tracking**: Added a lightweight, memory-backed token quota logic with an hourly reset interval (`QUOTA_LIMIT = 50000`) before calling the provider to defensively short-circuit when near rate limits.
- **Database Persistence**: Modified `schema.prisma` to add an `AIResult` model relational mapped to meetings. Successfully wired `prisma.aIResult.create` to store LLM responses upon generation, enabling database-backed auditability and reuse of summarizations instead of exclusively relying on the LRU memory cache.
- Synchronized tests with `PrismaClient` usage and ensured all 29 AI suite integration tests still pass successfully.

## Key Files Modified
- `backend/src/api/ai/ai.schema.js` (Added Output Zod schemas)
- `backend/src/services/ai.service.js` (Zod parsing, timeout, Quota Checks, Prisma persistence)
- `backend/prisma/schema.prisma` (Added `AIResult` table model)

## Security Additions
- Timeout guard prevents DoS through hung downstream AI providers.
- Quota limiting prevents unauthorized runaway token expenditure.
- DB logging persists request payloads allowing manual audit logs.

All changes verified via `npm test -- --testPathPattern="ai.test"` (Pass). All UAT acceptance checks established in `04-UAT.md` will continue to function seamlessly under these added structural guarantees.
